# 1.2 — Accordion Result List

## What

Replace the flat search result card list with an accordion that keeps one result
open at a time.

## Why

The current flat list becomes visually noisy with many results. Accordion
behavior keeps the overview scanable while preserving quick access to one active
result.

## Changes

- Update `templates/partials/search_results_partial.html` so search results are
  rendered as accordion items.
- Only one result card is expanded at a time.
- Collapsed state must still show:
  - section/table ID
  - title
  - code edition display name
  - page or page range
  - relevance signal if currently shown
- Expanded state must show the existing body content, including:
  - PDF block
  - HTML fallback
  - notes HTML
- Add keyboard-safe next/previous result controls if implemented in this task;
  if omitted, document that they remain for a later task and do not claim this
  task complete on that basis.

## Completion Conditions

- Search results are no longer rendered as a permanently expanded flat list.
- Opening one result collapses the previously open result.
- The first result is expanded by default after search results load.
- Collapsed summaries remain readable and do not hide result identity.
- Expanded result content still renders all existing fallback/UI branches.

## How to Test

### Template test: `test_search_results_partial_renders_accordion_structure`

Context:

```python
results = [
    {"id": "B-3.2.9.", "title": "Fire Separations", "code_display_name": "NBC 2025", "page": 120},
    {"id": "B-3.2.9.1.", "title": "Closures", "code_display_name": "NBC 2025", "page": 121},
]
```

Assertions:

```python
assert 'accordion' in rendered.lower() or 'x-data' in rendered
assert 'B-3.2.9.' in rendered
assert 'B-3.2.9.1.' in rendered
```

### Browser/manual test: `search_results_accordion_single_open`

Steps:

1. Run a search that returns at least 3 results.
2. Verify the first card is expanded initially.
3. Open the second card.
4. Verify the first card collapses.
5. Open the third card.
6. Verify the second card collapses.

Expected:

- Exactly one expanded card at any time.

### Browser/manual test: `search_results_accordion_preserves_fallback_content`

Run one search result with PDF and one with HTML fallback only.

Expected:

- Expanding the PDF result still shows the PDF block.
- Expanding the fallback-only result still shows fallback HTML.
