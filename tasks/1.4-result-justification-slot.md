# 1.4 — Result Justification Slot

## Status: implemented

## What

Reserve a stable location in the result card for future justification content
without implementing scoring explanations yet.

## Why

Later work may expose why a result ranked well. The card layout should have a
predictable place for that content before the more complex justification logic
exists.

## Changes

- Add a lightweight, collapsible slot/section in the expanded result card for
  future justification content.
- Do not implement keyword-frequency or scoring explanation logic in this task.
- The placeholder must not dominate the result card when empty.

## Completion Conditions

- Expanded result cards contain a clearly identified justification slot.
- Empty placeholder state does not create large blank space.
- No backend or scoring work is added beyond the UI shell.
