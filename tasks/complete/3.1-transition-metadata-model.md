# 3.1 — Transition Metadata Model

## Status: implemented

## What

Create the first transition metadata source for overlapping edition validity.

## Why

Edition overlap behavior should be driven by explicit curated metadata, not
implicit date hacks scattered through search logic.

## Changes

- Create `config/transitions.json`.
- Define Tier 1 transition fields:
  - `old_edition`
  - `new_edition`
  - `overlap_start`
  - `overlap_end`
  - `transition_type`
  - `applicability_text`
  - `citation_text`
- Add validation/loading code for this file.
- Reject malformed transition records loudly in tests.

## Completion Conditions

- `config/transitions.json` exists and loads successfully.
- Validation rejects missing required fields.
- Transition records are available to overlap-aware search code.

## How to Test

### Unit test: `test_load_transitions_returns_records_with_required_fields`

Assertions:

```python
records = load_transitions()
assert records
assert set(records[0]).issuperset({
    "old_edition", "new_edition", "overlap_start", "overlap_end",
    "transition_type", "applicability_text", "citation_text",
})
```

### Unit test: `test_load_transitions_rejects_missing_required_field`

Setup: temp JSON missing `citation_text`

Assertions:

```python
with pytest.raises(ValueError):
    load_transitions(temp_path)
```
