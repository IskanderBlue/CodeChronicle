# 4.3 — Viewer Query Context Emphasis

## Status: implemented, browser verification pending

## What

Preserve query context visually inside viewer mode even after the user starts
browsing.

## Why

Users need to know which edition/result originally matched their query when they
move into broader code browsing.

## Changes

- Show the query date and query-matching edition clearly in viewer mode.
- Keep dates visible when browsing other editions.
- Label non-query-matching editions as browse context.

## Completion Conditions

- Viewer mode always shows which edition matched the query.
- Query date remains visible.
- Browsed editions are not mistaken for the original match.

## How to Test

### Browser/manual test: `viewer_retains_query_context_after_switching_editions`

Expected:

- After switching editions, the viewer still indicates the original matching
  edition/date.

## Depends On

- 4.2 viewer edition navigation.

## Notes

- This is emphasis/labeling work, not navigation mechanics.
