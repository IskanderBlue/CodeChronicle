# 5.4 — Harden Static Asset Versioning/Caching

## Status: planned

## What

Switch Django's static file storage to `ManifestStaticFilesStorage` so that
collected files get content-hash suffixes (e.g., `tailwind.a1b2c3d4.css`).
Update nginx cache headers to use immutable long-lived caching for hashed
assets, and confirm Cloudflare respects origin cache headers for `/static/*`.

## Why

Currently nginx serves `/static/` with `expires 30d`, but filenames never
change between deploys. If a user or Cloudflare edge has a cached copy of
`tailwind.css`, they won't pick up changes until the 30-day TTL expires (or
they hard-refresh). With hashed filenames, every deploy that changes a file
produces a new URL — instant cache busting with no manual intervention.

## Current State

- `collectstatic` runs at container start via `scripts/entrypoint.sh` — **done**.
- `STATIC_ROOT = BASE_DIR / "staticfiles"` and `STATICFILES_DIRS` configured in
  `base.py` — **done**.
- Shared Docker volume `staticfiles` mounted into nginx at `/staticfiles/` —
  **done**.
- `ManifestStaticFilesStorage` — **not configured** (using default unhashed
  storage).
- nginx `expires 30d` on `/static/` — works but is unsafe without hashed names.
- Cloudflare caching behaviour for `/static/*` — **unverified**.

## Steps

### 1. Enable ManifestStaticFilesStorage in production settings

Add to `code_chronicle/settings/production.py`:

```python
STORAGES = {
    "staticfiles": {
        "BACKEND": "django.contrib.staticfiles.storage.ManifestStaticFilesStorage",
    },
}
```

This makes `collectstatic` write a `staticfiles.json` manifest mapping original
names to hashed names. The `{% static %}` tag then resolves to the hashed path.

Do **not** add this to `development.py` — dev uses the CDN and Django's default
static serving, which doesn't need hashing.

### 2. Update nginx cache headers

In `nginx/nginx.conf`, change the `/static/` location to use immutable caching
now that filenames are content-addressed:

```nginx
location /static/ {
    alias /staticfiles/;
    access_log off;
    expires 1y;
    add_header Cache-Control "public, immutable";
}
```

`immutable` tells browsers not to revalidate — safe because a content change
means a new filename.

### 3. Verify Cloudflare respects origin headers

Cloudflare's default behaviour is to respect origin `Cache-Control` headers for
static file extensions (.css, .js, .png, etc.). Confirm:

- Cloudflare is **not** overriding `/static/*` with a Page Rule or Cache Rule
  that sets a shorter TTL.
- The "Cache Level" for these paths is "Standard" (default), which defers to
  origin headers.

No code change needed unless a conflicting rule is found.

### 4. Handle `collectstatic` failure gracefully

`ManifestStaticFilesStorage` raises `ValueError` if a referenced file is
missing during post-processing (e.g., a CSS `url()` pointing to a nonexistent
image). Run `collectstatic` locally before merging to catch any issues:

```bash
python manage.py collectstatic --noinput --clear
```

Fix any broken references that surface.

## Risks

- **Breaking on missing references**: `ManifestStaticFilesStorage` is strict.
  If any CSS/JS file references a static asset that doesn't exist, `collectstatic`
  will fail. The Tailwind output (`tailwind.css`) is self-contained so this
  should be fine, but test with a local production-settings `collectstatic` first.
- **Volume timing**: `collectstatic` runs in the web container's entrypoint,
  and nginx reads from the same shared volume. There's a brief window during
  container startup where nginx may serve stale files. This is the existing
  behaviour and isn't made worse by this change — the hashed filenames actually
  help because old URLs remain valid until the manifest updates.

## Completion Conditions

- `collectstatic` with production settings produces hashed filenames in
  `staticfiles/` and a `staticfiles.json` manifest.
- `{% static 'css/tailwind.css' %}` resolves to a hashed URL in rendered HTML.
- nginx serves hashed assets with `Cache-Control: public, immutable` and a
  1-year expiry.
- A deploy that changes `tailwind.css` produces a different hashed filename.
- Dev environment is unaffected (no manifest storage, CDN still used).

## How to Test

### Local smoke test: `collectstatic produces manifest`

```bash
DJANGO_SETTINGS_MODULE=code_chronicle.settings.production \
SECRET_KEY=test DATABASE_URL=sqlite:///dev/null \
ALLOWED_HOSTS=localhost \
python manage.py collectstatic --noinput --clear --dry-run
```

Confirm output mentions writing `staticfiles.json`.

### Deploy test: `hashed URL in page source`

After deploying, view page source on production. The `<link>` tag should
reference something like `css/tailwind.a1b2c3d4.css`, not `css/tailwind.css`.

### Cache header test: `curl -I`

```bash
curl -I https://app.codechronicle.ca/static/css/tailwind.<hash>.css
```

Confirm response includes `Cache-Control: public, immutable` and a far-future
`Expires`.

## Depends On
