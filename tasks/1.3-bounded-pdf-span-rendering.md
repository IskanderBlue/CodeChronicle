# 1.3 — Bounded PDF Span Rendering

## What

Render each result's PDF from the top of the returned article/table to the
bottom of that returned article/table, rather than rendering a page with a bbox
highlight overlay.

## Why

The returned article/table itself is the focus. CC does not need interior
highlighting; it needs result-bounded PDF display.

## Changes

- Update result formatting and template attributes to pass:
  - `page`
  - `page_end`
  - `initial_page_top`
  - `final_page_bottom`
- Remove `bbox` from result formatting and template data attributes.
- Update `templates/search.html` PDF rendering logic:
  - first page: clip from `initial_page_top` downward
  - intermediate pages: render full height
  - last page: clip at `final_page_bottom`
  - single-page result: apply both top and bottom clipping on the same page
- Remove bbox highlight overlay creation.
- Preserve current PDF fallback behavior when the file is not available.
- If one or both span bounds are missing, render the full page or full page
  range instead of erroring.

## Completion Conditions

- No result rendering code depends on `bbox`.
- A single-page result renders only the bounded article/table span.
- A multi-page result renders first-page clip, full middle pages, and final-page
  clip.
- Missing boundaries do not break rendering.
- PDF fallback behavior still works when a PDF is unavailable or mismatched.

## How to Test

### Unit test: `test_format_search_results_emits_span_fields`

Input:

```python
raw = [{
    "id": "B-3.2.9.",
    "title": "Fire Separations",
    "code_edition": "NBC_2025",
    "page": 120,
    "page_end": 122,
    "initial_page_top": 640.0,
    "final_page_bottom": 88.0,
    "score": 1.0,
}]
```

Assertions:

```python
item = format_search_results(raw)[0]
assert item["page"] == 120
assert item["page_end"] == 122
assert item["initial_page_top"] == 640.0
assert item["final_page_bottom"] == 88.0
assert "bbox" not in item
```

### Template test: `test_search_results_partial_uses_span_data_attributes`

Assertions:

```python
assert 'data-pdf-page="120"' in rendered
assert 'data-pdf-page-end="122"' in rendered
assert 'data-pdf-initial-page-top="640.0"' in rendered
assert 'data-pdf-final-page-bottom="88.0"' in rendered
assert 'data-pdf-bbox' not in rendered
```

### Browser/manual test: `single_page_result_is_clipped_to_returned_span`

Setup:

- Result with `page == page_end == 120`
- `initial_page_top = 640`
- `final_page_bottom = 88`

Expected:

- The rendered PDF block shows only the returned section/table span.
- No visible bbox overlay rectangle appears.

### Browser/manual test: `multi_page_result_shows_full_middle_pages`

Setup:

- Result with `page = 120`, `page_end = 122`

Expected:

- Page 120 starts at the stored top bound.
- Page 121 is full height.
- Page 122 stops at the stored bottom bound.

### Browser/manual test: `missing_span_bounds_falls_back_cleanly`

Setup:

- Result with `page = 120`, `page_end = 121`, null top/bottom fields

Expected:

- Result renders full page 120 and 121.
- No JS errors in console.
