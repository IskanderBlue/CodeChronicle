# 3.2 — Overlap-Aware Edition Resolution

## What

Extend search orchestration so transitions can attach comparison results from an
older edition when the queried date falls inside an overlap window.

## Why

Users need to see both the primary edition result and the overlap edition result
when a grace period or transition makes both relevant.

## Changes

- Keep effective-date edition selection as the primary search path.
- After primary search, check active transitions for the queried date.
- If a transition applies, fetch secondary results from the older edition.
- Attach a `transition_context` object to affected results.
- When the same logical section appears in both editions, preserve enough data
  for Phase 3.3 to merge and compare them.

## Completion Conditions

- Searches outside overlap windows behave exactly as before.
- Searches inside overlap windows return both primary and comparison edition
  results when matching nodes exist.
- Each comparison-capable result includes transition context explaining why it
  appeared.

## How to Test

### Unit test: `test_execute_search_returns_primary_results_only_outside_overlap_window`

Assertions:

```python
response = execute_search({... date outside overlap ...})
assert all("transition_context" not in r for r in response["results"])
```

### Unit test: `test_execute_search_adds_overlap_results_inside_transition_window`

Assertions:

```python
response = execute_search({... date inside overlap ...})
matched = [r for r in response["results"] if r["id"] == "B-3.2.9."]
assert len(matched) == 2
assert {r["code_edition"] for r in matched} == {"BCBC_2024", "BCBC_2018"}
```

### Unit test: `test_overlap_results_include_transition_context_fields`

Assertions:

```python
ctx = matched[0]["transition_context"]
assert ctx["transition_type"] == "grace_period"
assert "citation_text" in ctx
assert "applicability_text" in ctx
```
