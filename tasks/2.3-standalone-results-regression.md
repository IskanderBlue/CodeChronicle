# 2.3 — Standalone Results Regression

## What

Preserve the simple standalone result path for hits that should not be grouped.

## Why

Grouping should improve dense sibling cases, not complicate ordinary one-off
results.

## Changes

- Ensure the formatter/frontend continue to render standalone results exactly as
  standalone cards when grouping conditions are not met.
- Confirm mixed result sets can contain both grouped and standalone items.

## Completion Conditions

- Results with no usable parent/child grouping remain standalone.
- Mixed grouped + standalone result sets render correctly in one response.
- Standalone results still use Phase 1 PDF span rendering.

## How to Test

### Unit test: `test_formatter_preserves_standalone_result_when_no_grouping_applies`

Assertions:

```python
assert len(results) == 1
assert results[0].get("group_type") is None
assert results[0]["id"] == "Table-9.10.3.1.-A"
```

### Template test: `test_mixed_grouped_and_standalone_results_render_together`

Assertions:

```python
assert 'B-3.2.9' in rendered  # grouped parent
assert 'Table-9.10.3.1.-A' in rendered  # standalone item
```
