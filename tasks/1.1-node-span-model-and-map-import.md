# 1.1 — Node Span Model and Map Import

## Status: implemented, pending DB-backed verification

## What

Replace CC's `bbox`-based node geometry with result-span fields that match the
PDF rendering behavior we actually want.

## Why

CC does not want interior highlight overlays. It wants to render the returned
article/table from its first-page top boundary to its last-page bottom
boundary. The persisted model should store exactly that.

## Changes

- Add the following nullable fields to `core.models.CodeMapNode`:
  - `initial_page_top`
  - `final_page_bottom`
- Keep:
  - `page`
  - `page_end`
  - `html`
  - `notes_html`
  - `parent_id`
  - `keywords`
- Deprecate and remove `bbox` from `CodeMapNode` and from all CC code paths.
- Create and apply a Django migration for the schema change.
- Update `core/management/commands/load_maps.py` so imported nodes are projected
  from mapping data into CC-facing span data:
  - `page` = first relevant page
  - `page_end` = last relevant page
  - `initial_page_top` = top boundary on first page
  - `final_page_bottom` = bottom boundary on last page
- Do not persist raw upstream `bboxes` in CC.
- If mapping data does not provide usable boundaries, leave top/bottom null and
  still import the node.

## Coordinate Convention

- Store `initial_page_top` and `final_page_bottom` in the same coordinate system
  already used by CC's PDF.js rendering code:
  - PDF points
  - origin at bottom-left
  - larger Y = higher on page

## Completion Conditions

- `CodeMapNode` no longer has a `bbox` field.
- `CodeMapNode` has `initial_page_top` and `final_page_bottom` fields.
- `load_maps` imports nodes successfully from map payloads that include span
  geometry.
- `load_maps` still imports nodes when geometry is missing.
- No remaining CC code reads or writes `CodeMapNode.bbox`.

## How to Test

### Unit test: `test_load_maps_projects_multi_page_span_fields`

Fixture payload:

```python
map_payload = {
    "code_name": "NBC_2025",
    "sections": [
        {
            "id": "B-3.2.9.",
            "title": "Test Section",
            "page": 120,
            "page_end": 122,
            "initial_page_top": 640.0,
            "final_page_bottom": 88.0,
            "html": "<p>Test</p>",
            "keywords": ["fire", "separation"],
        }
    ],
}
```

Assertions:

```python
node = CodeMapNode.objects.get(node_id="B-3.2.9.")
assert node.page == 120
assert node.page_end == 122
assert node.initial_page_top == 640.0
assert node.final_page_bottom == 88.0
assert node.html == "<p>Test</p>"
```

### Unit test: `test_load_maps_allows_missing_span_bounds`

Fixture payload:

```python
map_payload = {
    "code_name": "NBC_2025",
    "sections": [
        {"id": "B-3.2.9.1.", "title": "No Bounds", "page": 120, "page_end": 120}
    ],
}
```

Assertions:

```python
node = CodeMapNode.objects.get(node_id="B-3.2.9.1.")
assert node.page == 120
assert node.page_end == 120
assert node.initial_page_top is None
assert node.final_page_bottom is None
```

### Unit test: `test_load_maps_imports_notes_html_without_bbox`

Assertions:

```python
assert node.notes_html == "<div>Notes</div>"
```

- Added `initial_page_top` and `final_page_bottom` to `CodeMapNode`.
- Added migration `core/migrations/0007_remove_codemapnode_bbox_and_more.py`.

  removing `bbox`, so existing rows are preserved.
- Updated `load_maps` to read `initial_page_top` / `final_page_bottom` directly
  and to fall back to legacy `bbox` input when present.

  and the no-bounds fallback path.
