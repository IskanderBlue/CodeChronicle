# 4.1 — Viewer Mode Shell

## Status: implemented, pending browser verification

## What

Create a separate full-page viewer mode for browsing code content beyond the
returned result.

## Why

Search result mode should stay focused on the returned article/table. Browse mode
is the explicit way to move beyond that result.

## Changes

- Add a viewer-mode route/view/state distinct from the search results accordion.
- Add a "Browse in context" entry point from result cards.
- Viewer mode loads code hierarchy content and opens at:
  - result `page`
  - positioned to `initial_page_top`
- Add visible return/close behavior.
- Browser Back should return to search results.

## Completion Conditions

- User can enter viewer mode from a result card.
- Viewer mode opens at the clicked result's page/top position.
- User can exit back to results without losing context.

## How to Test

### Browser/manual test: `browse_in_context_opens_viewer_at_result_page_and_top`

Expected:

- Clicking "Browse in context" opens viewer mode.
- Viewer starts on the selected result's page.
- Initial scroll position matches the stored top boundary.

### Browser/manual test: `viewer_mode_close_returns_to_results`

Expected:

- Close button returns to the results page.
- Browser Back also returns to results.

## Depends On

- 1.3 bounded PDF span rendering.

## Notes

- Viewer mode is for browsing beyond the returned result, not for replacing the
  search card experience.

## Implementation Update (2026-03-08)

- Added a new viewer route at `core:viewer_mode`.
- Added `viewer_mode(...)` in `core/views/search.py` plus supporting query-param
  parsing for viewer entry.
- Added a `Browse in context` entry point to the shared result document block so
  standalone and compare cards can both open viewer mode.
- Added `templates/viewer_mode.html` as a full-page viewer shell with a close
  action that prefers browser-back behavior.
- Reused bounded PDF span rendering in viewer mode so the initial page/top entry
  matches search-result rendering expectations.
