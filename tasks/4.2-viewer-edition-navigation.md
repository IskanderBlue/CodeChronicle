# 4.2 — Viewer Edition Navigation

## What

Add previous/next edition navigation inside viewer mode.

## Why

Once users are browsing code context, they should be able to move across
editions without returning to search.

## Changes

- Add previous/next edition controls in viewer mode.
- Make it explicit when the viewed edition differs from the query-matching one.
- Do not implement split-view page alignment between editions.

## Completion Conditions

- Viewer mode can move to adjacent editions of the same code system.
- Query-matching edition is labeled/emphasized.
- Non-query-matching edition is clearly marked as browse context.

## How to Test

### Browser/manual test: `viewer_previous_next_edition_changes_displayed_edition`

Expected:

- Previous/next control changes edition.
- Edition label updates.

### Browser/manual test: `viewer_marks_non_query_matching_edition_as_browse_context`

Expected:

- Visual label indicates browse-context edition.

## Depends On

- 4.1 viewer mode shell.

## Notes

- Do not build side-by-side compare in viewer mode.
